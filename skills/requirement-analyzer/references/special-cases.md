# 特殊情况处理

## 转换需求

如果需求描述问题但实际是需求（如"XXX不工作"但期望是"希望XXX能工作"）:
- `is_bug` = true（标记为Bug）
- `is_feature_request` = true（同时标记为功能需求）
- 在 `core_value` 中说明实际意图

## 多需求混合

如果单个帖子包含多个独立需求:
- 在 `title` 中添加 `[含多个需求]` 标记
- 在 `core_value` 中列出包含的具体需求

## 处理规则

| 情况 | 处理方式 |
|------|----------|
| 信息不足 | 模块="未知"，复杂度="未知"，core_value标注"建议补充" |
| 内容理解困难 | 标记并说明需要人工审核 |
| JSON格式错误 | 尝试提取可解析部分，继续处理 |
| 模块无法确定 | 标注"其他" |
