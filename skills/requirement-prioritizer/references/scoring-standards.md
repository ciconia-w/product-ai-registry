# 评分标准详解

## 用户影响力（0-10分）

**评分公式**: `得分 = 用户覆盖 × 0.4 + 使用频率 × 0.4 + 核心程度 × 0.2`

| 得分 | 说明 | 判断依据 |
|------|------|----------|
| 9-10 | 几乎所有用户，核心功能 | 日常高频、核心模块 |
| 7-8 | 大部分用户，重要功能 | 中高频、重要体验 |
| 5-6 | 部分用户，次要功能 | 特定场景、低频 |
| 3-4 | 少数用户，边缘功能 | 很少人用、非必需 |
| 0-2 | 极少数用户，可选功能 | 小众需求 |

## 紧急程度（0-10分）

| 得分 | 说明 | 判断依据 |
|------|------|----------|
| 9-10 | 严重Bug，阻塞核心 | 系统崩溃、无法使用 |
| 7-8 | 重要Bug，影响核心 | 功能失效、重大问题 |
| 5-6 | 中等问题，部分影响 | 功能异常、一般问题 |
| 3-4 | 轻微问题，影响有限 | 小Bug、边缘问题 |
| 0-2 | 功能需求，非紧急 | 新功能、改进、优化 |

## 实现成本（原始评分0-10，最终得分 = 10-原始）

| 原始评分 | 说明 | 判断依据 |
|----------|------|----------|
| 0-2 | 成本极高 | 架构变更、多模块联动、长期项目 |
| 3-4 | 成本较高 | 复杂功能、需要重构 |
| 5-6 | 成本中等 | 单模块、有挑战 |
| 7-8 | 成本较低 | 简单功能、低风险 |
| 9-10 | 成本极低 | UI调整、配置、快速修复 |

## 战略价值（0-10分）

| 得分 | 说明 | 判断依据 |
|------|------|----------|
| 9-10 | 核心战略需求 | 核心目标、关键差异化 |
| 7-8 | 重要战略需求 | 符合方向、有差异化 |
| 5-6 | 一般战略价值 | 一致但非核心 |
| 3-4 | 较低战略价值 | 略有偏离 |
| 0-2 | 无战略价值 | 不符战略 |

## 依赖关系（0-10分）

| 得分 | 说明 | 判断依据 |
|------|------|----------|
| 9-10 | 关键前置需求 | 多个需求依赖它 |
| 7-8 | 重要前置需求 | 部分需求依赖它 |
| 5-6 | 有依赖关系 | 被少数需求依赖 |
| 3-4 | 依赖others | 依赖其他需求 |
| 0-2 | 无依赖关系 | 独立需求 |

## 社区热度（0-10分）

| 得分 | 说明 | 判断依据 |
|------|------|----------|
| 9-10 | 热度极高 | 点赞>50 或 回复>20 或 浏览>5000 |
| 7-8 | 热度高 | 点赞>30 或 回复>10 或 浏览>2000 |
| 5-6 | 热度中等 | 点赞>10 或 回复>5 或 浏览>500 |
| 3-4 | 热度较低 | 点赞>5 或 回复>2 或 浏览>100 |
| 0-2 | 热度低 | 低于上述阈值 |

**热度计算公式**: `热度 = min(10, 点赞数/5 + 回复数/2 + 浏览数/500)`

## 评分计算公式

```
优先级得分 =
  用户影响力 × 0.30 +
  紧急程度   × 0.20 +
  实现成本   × 0.15 +  (反向: 得分 = 10 - 原始评分)
  战略价值   × 0.15 +
  依赖关系   × 0.10 +
  社区热度   × 0.10
```

## AI 评分原则

1. **基于证据**: 从需求内容和元数据中找到支持评分的证据
2. **保持一致**: 相似类型需求应用相似评分标准
3. **相对比较**: 在整体需求集合中考虑相对重要性
4. **记录理由**: 为每个评分提供简短理由，便于审核
