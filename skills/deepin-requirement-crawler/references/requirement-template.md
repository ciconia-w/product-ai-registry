# 需求清单模板引用

本目录包含需求清单模板的引用和说明。

## 模板位置

**主模板**: 使用当前 capability 自带模板或宿主环境已配置的等价需求清单模板，不要硬编码本机绝对路径。

## 模板格式

```
# 需求清单

## 一、基线信息

| 项目 | 内容 |
|------|------|
| 产品名称 | （填写产品名称） |
| 支持的操作系统 | (Deepin V25, UOS V20, Windows 11) |
| 支持的CPU架构 | (AMD64, ARM64, LoongArch) |
| 目标用户 | (企业IT管理员, 普通员工) |
| 产品形态 | (桌面应用, Web应用, 移动应用) |

## 二、需求清单

### 模块名称：[模块1名称]

### 用户故事 1
- **作为**：（什么角色）
- **我希望**：（做什么操作或看到什么）
- **以便于**：（达到什么目的）
- **补充说明**：（可选，更多细节、限制条件等）

### 用户故事 2
- **作为**：（什么角色）
- **我希望**：（做什么操作或看到什么）
- **以便于**：（达到什么目的）
- **补充说明**：（可选）
```

## 使用说明

生成需求清单时，**严格遵循**上述模板格式。

## 字段填充规则

### 基线信息

| 字段 | 默认值/推断规则 |
|------|---------------|
| 产品名称 | 根据需求主题推断（如"Deepin 文件管理增强"） |
| 产品类型 | "新产品"（默认） |
| 支持的操作系统 | "Deepin V25"（默认） |
| 支持的CPU架构 | "AMD64, ARM64"（默认） |
| 目标用户 | "Deepin 用户"（默认） |
| 产品形态 | "桌面应用"（基于系统需求推断） |

### 需求清单

每个模块包含多个用户故事，用户故事包含四个字段：

| 字段 | 来源 |
|------|------|
| **作为** | 从需求分析的用户角色提取 |
| **我希望** | 需求标题转换为"我希望..."格式 |
| **以便于** | 从需求分析的核心价值提取 |
| **补充说明** | 需求的详细内容 |
